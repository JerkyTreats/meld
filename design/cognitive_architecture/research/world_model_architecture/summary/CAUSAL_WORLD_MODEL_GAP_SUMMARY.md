# Causal World Model Gap Summary

Source report: [Causal World Models for a Typed-Anchor Temporal Knowledge Graph](../pdf/Causal%20World%20Models%20for%20a%20Typed-Anchor%20Temporal%20Knowledge%20Graph.pdf)

Prompt source: [Causal World Models](../../research_prompts.md#3-causal-world-models)

## Definitions

- **Causal world model:** A world model that represents not only what happened after what, but which mechanisms caused which outcomes under actions, interventions, contexts, and counterfactual alternatives.
- **Structural causal model (SCM):** A causal model where each variable is generated by a structural assignment from its parent variables and exogenous noise. SCMs support intervention and counterfactual queries.
- **Endogenous variable:** A variable generated inside the model by other modeled variables, such as frame quality, task outcome, anchor selection, or post-task state.
- **Exogenous variable:** A source of external noise or hidden context outside the modeled causal system. Exogenous variables often represent unobserved causes.
- **Structural assignment:** A mechanism that defines how a variable is produced from its parents and noise, usually written as `X := f(parents, noise)`.
- **Mechanism:** The local causal process that generates one variable. Causal modeling treats mechanisms as modular so an intervention can alter one mechanism without rewriting the whole system.
- **Intervention:** An action that changes a mechanism or variable, represented by `do(...)` in SCM notation.
- **Hard intervention:** An intervention that sets a variable directly and severs its normal incoming causes.
- **Soft intervention:** An intervention that changes a variable's generating mechanism without fully severing all incoming causes.
- **Unknown-target intervention:** An intervention where the system knows an action occurred but does not know which mechanism or variable it actually perturbed.
- **Observational data:** Data passively observed from the system. It can show correlation but may not identify causal effects.
- **Interventional data:** Data produced when a mechanism is deliberately manipulated. It can identify causal directions and effects that observational data cannot.
- **Confounder:** A hidden or observed cause of both an action and an outcome. Confounders can make a task appear causal when both task choice and outcome were actually driven by a third factor.
- **Sequential ignorability:** An assumption that, given observed history, task assignment is not driven by hidden causes that also affect the outcome. Logged adaptive agent actions often violate this unless policy state is fully observed.
- **Causal discovery:** Methods for learning causal graph structure from data, such as PC, GES/GIES, NOTEARS/DYNOTEARS, and DCDI.
- **Markov equivalence class:** A set of causal graphs that imply the same observational conditional independencies. Observational data often identifies only this class, not one unique graph.
- **Counterfactual query:** A query about what would have happened under a different action or context, such as whether frame quality would still have improved if task `X` had not run.
- **Regime or meta-state:** A context in which the causal graph or mechanisms change, such as a different domain, workflow phase, or latent operating mode.

## Prompt Gap

The prompt targeted this gap:

> The knowledge graph captures correlation-shaped anchors: when execution of task X completes, frame Y is selected. The belief layer will eventually need to reason about whether the quality of Y causally depends on X, or whether there is a confound.

The research question was whether a typed-anchor temporal graph can become causal just by adding richer edge labels, or whether it needs a deeper semantic split between state, action, outcome, selection, and hidden context.

## Answer To The Gap

The report answers the gap by saying that causal reasoning requires a change in graph semantics, not merely a new relation vocabulary.

A correlational temporal graph can answer "what tended to happen after what." A causal world model must answer "what mechanism changed under which intervention, and would the outcome still have occurred under a different action or context?" That requires explicit variables for world state, task execution, outcome quality, selection policy, and regime or hidden context.

For Meld, the central answer is that "anchor selected pointing at frame Y" must not be treated as a causal fact by itself. It should usually be decomposed into at least:

- a world or system state variable
- a task execution or treatment variable
- a frame quality or task-success outcome variable
- an anchor selection or attention variable
- a regime or context variable when mechanisms can change

The anchor remains evidence about state, selection, or outcome. It should not stand in for the causal claim that the task caused the frame to be good.

## Prompt Questions Mapped To Report Answers

### From Temporal Graph To Causal Model

The report finds that the SCM/do-calculus tradition changes the meaning of the graph. Nodes should denote variables in a generative process, and edges should denote parent relationships in structural assignments. Events are not enough; the model also needs mechanisms, exogenous context, and interventions.

Answer: the causal upgrade is a dynamic SCM or interventional dynamic Bayesian network over time slices, not a temporal graph with extra causal labels.

### Task Executions As Interventions

The report gives a conditional answer. Task execution records can be treated as interventional only when the task actually manipulates a target mechanism and the task choice is exogenous enough for the causal question.

Answer: execution records should be modeled as candidate intervention contexts, not automatically as clean interventions. If task choice depends on hidden policy state or hidden world state, the log remains observational from the learner's perspective.

### Observational Versus Interventional Identification

Observational data can leave several explanations compatible with the same event history: task `X` improved frame quality, hidden context caused both task `X` and quality, or the anchor policy selected a frame because it was already good.

Answer: treating task execution as an explicit intervention node helps the system estimate action effects, ask counterfactuals, and localize confounding. It does not by itself remove confounding; it gives the model a place to represent and test it.

### Scalable And Changing Causal World Models

The recent literature does not converge on one production-ready causal world model for large, nonstationary, agentic systems. The recurring pattern is sparse, modular, object-factored, or regime-conditioned causal structure.

Answer: Meld should avoid assuming one fixed global DAG. It should expect local mechanisms, sparse causal neighborhoods, object or entity factorization, and multiple graph structures indexed by regime or meta-state.

### Representing Causal Uncertainty

The report finds that a posterior over graph structures is principled but insufficient. Interventional predictions depend on graph structure, mechanism parameters, intervention targets, intervention types, hidden confounders, and changing regimes.

Answer: the belief layer should represent causal uncertainty as a coupled uncertainty over structure, mechanisms, intervention targets, confounders, and regime, then expose posterior predictive answers to task-relevant intervention and counterfactual queries.

## Design Consequences

This report turns the causal gap into several concrete architecture requirements:

- Split the current typed-anchor graph into a causal state layer and an anchor or measurement layer.
- Model task executions as treatment/intervention variables with parameters, intended targets, and intervention type.
- Model frame quality, task success, downstream utility, and validation signals as outcome variables, not as properties implied by anchor selection.
- Model anchor selection as a policy or measurement process, including salience, attention, and policy state.
- Add explicit regime or context variables for cases where causal mechanisms change over time.
- Treat logged executions as candidate interventions until metadata or assumptions justify stronger claims.
- Preserve hidden-confounding risk in the belief state instead of converting correlation into causation.
- Expose causal queries such as "what is the posterior effect of executing task X on frame quality?" and "would anchor A still have been selected if task X had not run?"

## What Remains Open

The report leaves three major unresolved areas:

- Intervention semantics: a task attempt is not automatically a successful manipulation of the intended mechanism, especially when interventions can fail, partially apply, be delayed, or have unknown targets.
- Hidden confounding: adaptive execution logs can remain observational if task choice depends on hidden context.
- Scale under nonstationarity: current research improves scalability with sparsity, object factorization, and mixture-of-graphs approaches, but there is not yet a settled model for large, partially observed, asynchronous, dynamically changing agentic systems.

The practical conclusion is architectural: keep typed anchors as evidence and measurement, add explicit causal variables for state/action/outcome/selection/regime, and let the belief layer reason over intervention uncertainty rather than treating temporal precedence as causal proof.
